# Second-Coder Onboarding

## Purpose

This guide prepares a second coder to complete the ten-case TRIM pilot from the source packet and coding manuals.

## Materials

Use the current codebook, coding manuals, segmentation workflow, pilot protocol, source packet, case manifest, and coding template.

## Coding Sequence

1. Record evidence, anchor, function, friction locus, uncertainty, and rationale.
2. Complete the remaining signature fields and any alternative pathway.
3. Save the completed file before comparison.
4. Generate agreement and disagreement outputs.
5. Begin adjudication after the first comparison has been archived.

## Pilot Coverage

The sample contains four *Zuo zhuan* cases, three *Macbeth* cases, and three *In a Grove* cases. It covers five demonstrated friction-locus values, simple and compound mechanisms, three uncertainty levels, and one contested case.

The pilot evaluates manual usability, field boundaries, disagreement location, and three pre-specified comparative patterns.

## Returning Data

Return the completed CSV with a distinct `coder_id` and project `status`. Preserve the original file alongside any adjudicated version.

See `docs/intercoder_workflow.md` for comparison and reporting.
