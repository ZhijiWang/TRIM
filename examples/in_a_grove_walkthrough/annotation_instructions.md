# Annotation Instructions: In a Grove Walkthrough v0.1

Use only the TRIM-HAA Core fields:

- Primary evidence
- Function label
- Rationale mechanism
- Uncertainty
- Rationale note
- Alternative pathway present
- Alternative mechanism
- Alternative note

This walkthrough is an independent record audit. It is not an exposure audit.

## Evidence Rule

Background knowledge may inform interpretation, but every formal evidence claim must cite one or more packet segment IDs. External cultural claims must be explicitly marked as external and cannot substitute for textual evidence.

## Function Labels

Choose one label.

### `self_inflicted_death`

Plain-language definition: The record foregrounds the samurai's own account that he caused the fatal wound.

Inclusion rule: Use when the primary evidence is the samurai's statement that he took the small sword and stabbed himself.

Exclusion rule: Do not use if the record mainly foregrounds unresolved agency, mediation, or competing responsibility.

Packet example: `IAG-SAM-010`.

Boundary note: This label describes what the record foregrounds, not an adjudicated truth.

### `unresolved_agency`

Plain-language definition: The record foregrounds uncertainty or tension about who controls the final meaning of the death.

Inclusion rule: Use when the selected evidence includes both self-stabbing and a later or mediating element that keeps agency open.

Exclusion rule: Do not use for mere uncertainty without packet evidence that creates a competing pathway.

Packet example: `IAG-SAM-010` with `IAG-SAM-012`.

Boundary note: This does not prove that another person killed the samurai.

### `transferred_blame`

Plain-language definition: The record foregrounds how the testimony shifts moral blame toward another character.

Inclusion rule: Use when the selected evidence focuses on accusation, demand, shame, or responsibility attributed to another character.

Exclusion rule: Do not use when the record mainly describes physical cause of death.

Packet example: `IAG-SAM-003`.

Boundary note: Blame is not the same as physical agency.

### `testimony_mediation`

Plain-language definition: The record foregrounds the fact that the testimony is mediated through a medium.

Inclusion rule: Use when the selected evidence depends on the framing of the dead speaker's testimony rather than only on events described inside the testimony.

Exclusion rule: Do not use merely because the passage is from the medium section if mediation is not analytically used.

Packet example: `IAG-MED-001`.

Boundary note: Mediation is an evidential condition, not a diagnosis of falsity.

### `no_fit`

Plain-language definition: None of the available labels adequately describes the record.

Inclusion rule: Use only when the packet evidence does not support any listed label.

Exclusion rule: Do not use simply because two labels seem plausible; use uncertainty and alternative pathway if needed.

Packet example: none in this packet by design.

Boundary note: No-fit is a last-resort label.

## Rationale Mechanisms

Choose one mechanism.

### `direct_action`

Plain-language definition: The evidence supports the label because a speaker directly reports an action.

Inclusion rule: Use for action statements such as taking, stabbing, cutting, fleeing, or removing.

Exclusion rule: Do not use when the evidence matters mainly because it conflicts with another segment.

Packet example: `IAG-SAM-010`.

Boundary note: A direct report is still a report, not an adjudicated fact.

### `contrast_or_tension`

Plain-language definition: The evidence supports the label because two packet elements pull interpretation in different directions.

Inclusion rule: Use when the record depends on a tension such as self-stabbing plus a later unidentified intervention.

Exclusion rule: Do not use for simple sequence without interpretive tension.

Packet example: `IAG-SAM-010` with `IAG-SAM-012`.

Boundary note: Tension licenses a question, not a verdict.

### `mediation_frame`

Plain-language definition: The evidence supports the label because the testimony is framed as conveyed through a medium.

Inclusion rule: Use when the record relies on mediated testimony as part of the evidence.

Exclusion rule: Do not use when mediation is merely background and not part of the annotation.

Packet example: `IAG-MED-001`.

Boundary note: Mediation does not automatically make the testimony unreliable.

### `moral_attribution`

Plain-language definition: The evidence supports the label by assigning blame, shame, or responsibility.

Inclusion rule: Use when the selected evidence is about accusation or moral burden.

Exclusion rule: Do not use for physical action alone.

Packet example: `IAG-SAM-003`.

Boundary note: Moral responsibility and physical cause can diverge.

### `omission_or_gap`

Plain-language definition: The evidence supports the label because something important remains unspecified.

Inclusion rule: Use when an unidentified actor, missing motive, or unstated transition is the key support.

Exclusion rule: Do not use for ordinary ambiguity that has no specific packet anchor.

Packet example: `IAG-SAM-012`.

Boundary note: A gap invites further evaluation; it does not establish the missing fact.
