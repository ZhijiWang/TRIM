# TRIM-HAA Practice-Case Protocol

The practice case is not scored for interpretive correctness. It tests whether the participant can use the fields well enough to proceed with formal pilot data collection.

Expected session length: approximately 60-90 minutes, including instructions, practice, breaks, annotation tasks, and feedback questions. This duration range is provisional and must be finalised before ethics submission and recruitment.

## Practice Comprehension Criteria

The participant must show that they can:

1. Select at least one primary-evidence segment.
2. Distinguish function label from rationale mechanism, meaning the connection between evidence and label.
3. Choose an uncertainty level.
4. Write a short rationale note.
5. Understand that an alternative pathway is optional.
6. Understand that AI output, when later shown, is not an answer key.

## Standardised Procedure

1. First attempt: participant completes the practice case using the written guides.
2. Permitted procedural clarification: researcher may repeat written field definitions and interface instructions.
3. Standardised re-explanation: if needed, researcher repeats the approved synthetic practice example only.
4. Optional second practice attempt: participant may complete one additional practice attempt after re-explanation.
5. Formal-pilot eligibility decision: researcher records one of the outcomes below.

## Practice Outcomes

- `ready_for_formal_pilot`: participant completes the practice case and meets the criteria without extra support beyond ordinary instructions.
- `ready_with_documented_support`: participant meets the criteria after permitted clarification or standardised re-explanation.
- `usability_feedback_only`: participant does not meet the criteria, but consent permits retaining usability feedback.
- `stop_session`: participant stops or the researcher stops the session under the stopping rules.

If a participant cannot distinguish function label from rationale mechanism after standardised re-explanation, do not proceed with formal pilot data collection. Retain only usability feedback if consent permits. Record the issue as instrument failure, not participant failure.
