# Changelog

## 0.2.0 - Unreleased

### Added

- Source segmentation and second-coder workflows.
- Coding manuals, methodological documentation, and article-use guidance.
- Ten-case demonstration interpretations and blinded pilot materials.
- Compound-aware intercoder metrics and optional kappa support.

### Changed

- Defined interpretive friction as a relational conversion difficulty.
- Clarified threshold location, epistemic support, and dominant-threshold selection.
- Documented the empirical status of each friction-locus value.
- Renamed generated `interpretive_payoff` output to `comparison_prompt`.
- Updated source and citation metadata to 0.2.0.
- Strengthened evidence, anchor, signature, and contested-case validation.
- Rewrote public documentation as a unified methodological argument.

### Removed

- Tracked macOS AppleDouble files and stale internal readiness reports.

## v0.1.1

TRIM v0.1.1 established the initial public package for structured humanistic annotation, including controlled vocabularies, signature parsing, comparison utilities, graph export, intercoder preparation, a ten-case demonstration corpus, coding documentation, citation metadata, and test coverage.
